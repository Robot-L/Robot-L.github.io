如何优雅的进行批量业务校验

发表于 2019-12-18 分类于 亚信 ， AppFrame 阅读次数： Valine：

数据校验分为为两种，一是基本数据校验，如参数非空校验，枚举值合法校验；二是业务校验，需要根据业务场景设计校验规则，我们这里只讨论批量的业务数据校验，例如用户在页面上选择一批号码数据进行删除，规则为：

1.号码必须在数据库中存在

2.号码的状态必须为“未下发状态”

基本数据校验不通过，属于调用接口传参的问题，应该直接抛出异常。而对于批量业务校验，应该有详细的错误信息，批量的业务校验要将校验失败的数据和对应的错误原因返回给前端进行处理。

通常我们在代码里的校验会用if语句和for循环来处理这些校验，但是代码很繁琐，下面介绍一些更优雅的处理方式。

二、批量业务校验

对于批量数据的业务校验，其实整理一下校验的业务场景，可以归纳为4类：

  1. 存在性校验(exist)，比如资源删除业务，要校验数据库中资源是否存在，不存在的话不能执行删除业务

  2. 不存在校验(not-exist)，比如资源生成业务，要校验数据库中资源是否不存在，如果存在的话不能执行生成业务

  3. 相等校验(equal)，某个属性必须与某值（或某些值）相等，如号码下发时号码状态必须是1-空闲

  4. 不相等校验(not-equal)，某个属性必须不等于某值（或某些值），如一卡多号号码延期业务，号码必须是下发之后的，所以号码状态必须不为1-空闲

一般简单的业务场景基本都是这4种校验的组合。
